# RAG 性能基准与验收

## 改造前基线

对“公司员工带薪年休假有几天？”的两次真实请求总耗时约 20.9 秒和 21.3 秒。其中检索为约 99–137ms，BGE 重排为约 19.7–20.2 秒，生成约 1 秒。因此主要瓶颈是每次在线问答都执行 CPU Cross-Encoder，而不是向量数据库。

## 改造后的验收口径

- embedding 预热后，检索和证据返回 P95 ≤ 1000ms。
- embedding 预热后，首个生成 token P95 ≤ 2000ms。
- 线上默认重排耗时必须为 0ms；如需开启，先在离线数据集上证明重排 P95 < 250ms。
- 质量基准至少记录 Recall@5、MRR/NDCG、引用准确率和无依据回答率。

## 本机验证（当前知识库）

在重新索引现有 61 个切片后，“员工带薪年休假有几天？”的首次进程检索为约 6588ms（BGE 模型冷加载），同一进程第二次检索为约 93ms；首条证据为年休假政策 PDF 第 1 页。证据详情、相邻切片和原文件下载映射均验证可用。生产进程通过启动预热消除这次冷加载。

## 回归样本

当前“员工带薪年休假”问题必须将政策 PDF 的正确页作为主要证据，不能把商品 Excel 作为首要来源。`tests/test_pure_rag_contract.py` 锁定切片引用、单次 RRF 和线上关闭重排；`tests/test_rag_api_contract.py` 锁定 SSE 证据先行与证据详情接口。
